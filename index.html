What It Does
Core Features
Everything an agent needs to run faster, serve better, and earn more — in one lightweight app.
⚡
USSD Automation
Receives M-Pesa SMS and automatically dials the USSD sequence for you. No manual input, no delays.
🧠
Advanced USSD Steps
Multi-step USSD flows with per-step responses, conditions, and fallback sequences when the first attempt fails.
🔀
Alternative USSD
Automatically tries an alternative USSD code when the primary flow fails — zero manual retry needed.
📡
SMS Relay
Forward incoming M-Pesa messages to partner agents via SMS. Run receiver and sender relay rules side by side.
🌐
Web Orders
Customers can order directly from your personal link — bingwaflash.co.ke/yourname — and payment triggers automation.
🤖
Offers Bot
Automated SMS bot that replies to customer inquiries, recommends offers, and collects recipient numbers before payment.
🕐
Scheduled Transactions
Schedule any transaction to run at a future time. Auto-retries are handled within a configurable window.
🚨
Agent Alerts
SMS notifications for key events — automation off, failed thresholds, reversals, and blacklisted customers.
🏪
Dual SIM Support
Choose which SIM slot executes each offer. Run different offers on different SIMs simultaneously.
📊
Transaction History
Full log of every transaction — status, USSD response, fail reason, and the original M-Pesa SMS. Searchable and filterable.
🚫
Blacklist & Offer Control
Block specific customers, disable individual offers, and prevent duplicate transactions automatically.
💾
Contact Auto-Save
Automatically saves paying customers to your phonebook with a custom suffix. Update, duplicate or skip existing contacts.
The Flow
How It Works
01
Customer Pays
A customer sends M-Pesa to your till or number. The app intercepts the SMS in real time.
02
Offer Matched
The amount and payment mode are matched against your configured offers. The correct USSD template is picked automatically.
03
USSD Executed
The app dials the USSD sequence silently using the Android accessibility service — no screen unlocking required.
04
Logged & Done
Result is logged with the USSD response. If it fails, the app retries or flags it for manual review.
